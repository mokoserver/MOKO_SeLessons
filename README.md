# MOKO SE Lessons - Учебные материалы по MOKO SE

## Описание

Этот репозиторий содержит коллекцию пошаговых уроков и примеров для изучения возможностей автоматизированной тестовой среды MOKO SE. Каждый урок представляет собой отдельный проект (`.msep`), состоящий из Python-скриптов и сопутствующих файлов, и демонстрирует конкретную функцию или аспект системы.

Эти материалы предназначены для новых пользователей, чтобы помочь им освоить процесс написания тестовых сценариев, работы с плагинами, драйверами и другими компонентами MOKO SE.

## Структура уроков

Репозиторий организован в виде набора папок, каждая из которых соответствует определённому уроку.

- **`Lesson 1-3`**: Введение в MOKO SE. Основы создания проекта, использование функции `MOKO.Stage` для структурирования теста.

- **`Lesson 4`**: Демонстрация функции `Messenger` для отображения информации и изображений в ходе выполнения теста.

- **`Lesson 5`**: Работа с отчётами. Показывает, как добавлять в отчёт текст, таблицы, изображения и другие данные с помощью функции `Report`.

- **`Lesson 6`**: Использование функции `Driver` для взаимодействия с измерительным оборудованием.

- **`Lesson 7`**: Работа с плагинами через функцию `Plugin`.

- **`Lesson 8`**: Использование внешних утилит (часто на LabVIEW) с помощью функции `Utility`.

- **`Lesson 9`**: Управление выполнением скриптов и проектов с помощью функции `Program`.

- **`Lesson 10`**: Интеграция с Telegram для отправки уведомлений с помощью модуля `MTLG`.

- **`Lesson 11`**: Примеры построения сложной архитектуры тестового проекта, включая регистрацию образца, выполнение тестов по методике и формирование протокола.

- **`Lesson 12`**: Углублённая работа с плагином `MOKO Graph` для построения и кастомизации графиков.

- **`Lesson 13`**: Демонстрация работы с плагином `MOKO Clicker` для эмуляции действий пользователя (клики, перемещение мыши, скриншоты).

- **`DU Lesson 1-2`**: Уроки по созданию собственных драйверов и утилит для расширения функциональности MOKO SE.

Каждый урок содержит Python-скрипты с комментариями, файл проекта MOKO SE (`.msep`) и, как правило, шаблон отчёта в формате `.docx`.
